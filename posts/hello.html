Hello, world!

This is a test post using a custom EDSL in Haskell for my blog. This will eventually become an EDSL for my Explorable Explanations!

Why an EDSL?

At first, I was thinking of creating a superset of Markdown to handle the stuff I wanted to do. This meant that I had to write a parser for this new DSL. For static pages, this wouldn't be so bad. However, with dynamic pages for explorable explanations, I would have to implement a basic programming language. It would be easier and simpler to rely on another language: in this case, Haskell.

I also tried jusing Dhall for "configuring" the site and pages to my liking. However, this was proving more difficult as pages lend themselves to a recursive style: you can have headers, sub-headers, sub-sub headers, etc. Also, it was becoming a pain-in-the-ass to actually write content and wouldn't handle Explorable Explanation functionality.

Instead, I took the halfway point of an EDSL, which sits below a DSL but also not a configuration-like thing. It would allow me to add more Explorable Explanation functionality later, which is my main goal of this blog.
